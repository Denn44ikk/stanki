# Architecture

## Overview

Проект состоит из четырех рабочих слоёв и одного legacy-слоя:

- `src/app` — frontend Vite/React. Здесь находится интерфейс менеджера: проекты, подтверждение AI-разбора, редактор схемы, ручные правки и экспорт.
- `server/http` — Express API. Это единственная точка входа для клиента.
- `server/ai` — интеграция с OpenRouter. Здесь живут реальные вызовы LLM.
- `server/persistence` — SQLite-хранилище проектов.
- `shared/domain` — общая доменная модель: каталог оборудования, `zod`-контракты, геометрия, layout-engine и fallback-правила.
- `public/dxf-machines` и `public/dxf-machines-preview` — runtime-ассеты станков: полная геометрия для экспорта и PNG-превью средней детализации для UI.
- `scripts/generate-machine-previews.mjs` — служебный генератор PNG-превью из полной библиотеки SVG.
- `src/legacy` — старый MVP ручной расстановки, оставленный как референс и отдельный тестируемый срез.

## Where AI Participates

AI участвует не во всём пайплайне, а в двух конкретных местах:

1. Разбор текста менеджера:
   - маршрут `POST /api/projects/:id/ai/parse`;
   - backend вызывает OpenRouter и просит вернуть структурированный JSON;
   - если ключа нет, используется детерминированный разбор из `shared/domain/project-engine.ts`.

2. Подготовка правок:
   - маршрут `POST /api/projects/:id/ai/edit`;
   - backend просит AI вернуть структурированный patch (`add/remove/replace/move/rotate`);
   - patch не применяется автоматически, сначала показывается пользователю.

## Why Coordinates Are Not Generated by AI

Это сделано специально. Координаты и геометрия строятся детерминированным движком из `shared/domain/project-engine.ts`, чтобы:

- схема была повторяемой;
- легче было тестировать коллизии и выходы за границы;
- AI не записывал произвольные `x/y` прямо в сохранённый проект.

То есть роль AI здесь — понять смысл менеджерского текста и предложить осмысленные изменения, а не заменить инженерную логику.

## Runtime Flow

1. Пользователь создаёт проект в `manual` или `ai` режиме.
2. Клиент обращается в `server/http`.
3. Для AI-сценариев сервер идёт в `server/ai/openrouter.ts`.
4. Ответ приводится к общим контрактам из `shared/domain/contracts.ts`.
5. Layout строится в `shared/domain/project-engine.ts`.
6. Проект сохраняется в SQLite через `server/persistence/project-store.ts`.
7. UI-превью берутся из `public/dxf-machines-preview`, а экспорт читает полные SVG из `public/dxf-machines`.
8. Экспорт идёт через `shared/exporters/dxf.ts` и `server/exports/pdf.ts`.
