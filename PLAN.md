## Goal

Собрать понятную full-stack систему для проектирования линии:
- ручной режим сборки состава и схемы;
- AI-режим разбора текста менеджера и подготовки правок;
- сохранение проектов;
- экспорт в DXF и PDF.

## Architecture / Structure

Текущая согласованная структура проекта:

```text
src/
  app/                 # рабочий frontend Vite/React
  legacy/              # предыдущий MVP и референсный код
  main.tsx             # клиентский entrypoint
  styles/              # глобальные стили
server/
  ai/                  # интеграции с LLM
  exports/             # серверные экспортеры
  http/                # Express entrypoints и маршруты
  persistence/         # SQLite-хранилище
shared/
  domain/              # схемы, каталог, геометрия, layout-engine
  exporters/           # общие CAD-экспортеры
tests/
  integration/         # API и end-to-end сценарии
  legacy/              # тесты старого MVP
  unit/                # модульные тесты новой доменной логики
docs/
  architecture.md      # краткое описание слоёв и AI-flow
PLAN.md
AGENTS.md
```

## Decisions

- AI не пишет координаты напрямую в проект: текст интерпретирует AI, координаты строит детерминированный layout-engine.
- Реальный LLM вызывается только на backend через OpenRouter.
- При отсутствии `OPENROUTER_API_KEY` система работает в fallback-режиме на детерминированных правилах и алиасах.
- Старый single-screen MVP сохраняется как `legacy`, чтобы не терять референсный код, но не участвует в основном приложении.
