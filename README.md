# AI-First Layout Constructor

Веб-приложение для сборки, AI-разбора и редактирования производственных схем линии.

Текущая версия уже работает как проектная система:
- есть `Ручной` и `AI` режимы;
- проекты сохраняются в `SQLite`;
- AI-разбор и AI-правки идут через backend;
- итог можно выгрузить в `DXF`, `PDF` и локальное `PNG` превью;
- геометрия и layout считаются детерминированным движком, а не напрямую моделью.

## Что теперь умеет приложение

- создавать и переоткрывать проекты;
- выбирать режим `manual` или `ai`;
- задавать размеры цеха явно;
- хранить текст менеджера, состав линии, шаблон, схему и историю AI-патчей;
- разбирать текст менеджера в подтверждаемый состав оборудования;
- подтверждать AI-состав перед построением схемы;
- собирать состав линии вручную через UI;
- строить схему по шаблонам и правилам;
- вручную двигать и поворачивать элементы на плане;
- запрашивать у AI структурированные правки схемы с preview перед применением;
- сохранять проект и экспортировать инженерные файлы.

### Основные слои

- `shared/contracts.ts` — общие `zod`-схемы и типы проекта.
- `shared/catalog.ts` — seed-каталог оборудования, шаблоны и few-shot примеры из `AI/_ИИ`.
- `shared/project-engine.ts` — разбор текста, materialize confirm step, layout engine, AI patch fallback.
- `server/store.ts` — хранение проектов в `SQLite`.
- `server/openrouter.ts` — интеграция с `OpenRouter`, если заданы env-переменные.
- `server/pdf.ts` и `shared/exporters/dxf.ts` — инженерные экспорты.

## Источник данных

Папка `AI/_ИИ` используется как референсный материал:
- для seed-каталога;
- для алиасов и сокращений;
- для шаблонов линий;
- для few-shot примеров при работе через OpenRouter.

Бинарные `CDW/DXF/PDF` из этой папки не импортируются автоматически во время работы приложения.

## Установка

```bash
npm install
```

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните при необходимости:

```bash
OPENROUTER_API_KEY=
OPENROUTER_MODEL=openai/gpt-4o-mini
OPENROUTER_HTTP_REFERER=http://localhost:5173
OPENROUTER_APP_TITLE=stanki
PORT=8787
STANKI_DB_PATH=./data/projects.sqlite
```

Если `OPENROUTER_API_KEY` не задан, приложение всё равно работает, но AI-сценарии уходят в детерминированный fallback по словарям и правилам.

## Пользовательский сценарий

### Ручной режим

1. Создайте проект в режиме `manual`.
2. Укажите размеры цеха.
3. Выберите шаблон или оставьте без шаблона.
4. Добавьте позиции из каталога.
5. Нажмите `Построить схему`.
6. Подвиньте или поверните элементы.
7. Сохраните проект.
8. Выгрузите `DXF` или `PDF`.

### AI-режим

1. Создайте проект в режиме `ai`.
2. Вставьте текст менеджера.
3. Нажмите `Разобрать AI`.
4. Проверьте предложенный состав, шаблон и unresolved tokens.
5. Подтвердите состав.
6. Получите автоматически построенную схему.
7. При необходимости внесите ручные правки.
8. При необходимости отправьте follow-up инструкцию в блок `AI-правки`.
9. Просмотрите `preview` патча.
10. Подтвердите применение патча.
11. Сохраните проект и выгрузите результат.

## Экспорт

### DXF

`DXF` формируется как ASCII-чертёж с секцией `ENTITIES` и содержит:
- контур помещения;
- контуры оборудования;
- контуры зон безопасности;
- базовые размерные линии;
- текстовые подписи элементов.

### PDF

`PDF` формируется на backend как инженерное представление схемы:
- план цеха;
- размещения оборудования;
- правый side panel с составом, статусом и предупреждениями.

### PNG

`PNG` сейчас остаётся локальным preview из canvas и нужен для быстрой визуальной проверки.

## Ограничения текущей версии

- каталог и шаблоны пока seed-ятся вручную из материалов `AI/_ИИ`, а не через отдельную админку;
- PDF использует безопасную ASCII-транслитерацию текста для совместимости со стандартными шрифтами;
- fallback AI-логика пока словарная и rule-based;
- нет авторизации и многопользовательского режима;
- нет полного коммерческого пакета с Excel/CDW-генерацией;
- `DXF` и `PDF` есть, но нативный `CDW` не генерируется.
